Ruby Variable Scope

What kinds of variables does Ruby support? Where are they accessible? When would you use them and what makes them different?

Local Variables

Local variables are declared inside of blocks and are scrapped to trash collector at the end of the block. These stay only as long as they are told. You can tell a local variable's scope by following a few rules.
The top level (outside of all definition blocks) has its own local scope. Every class or module definition block (class, module) has its own local scope, even nested class/module definition blocks. Every method definition (def) has its own local scope.

Global Variables

Global Variable begin with $. Uninitilaized global variables default to NIL. These are not recommended as the make a program cryptic but they are accessible from anywhere. You use global variables when you want one number to be constant throught the entire life of the product.

Class Variables

Class variables are created with @@ prefix. This should be used when there is only one value that exists for all objects of the given class. It is a global variable within the context of the class.

Instance Variables

Instance variables begin with @ and also initialize to nil. They are instantiated every time a new object of the type is created. This is often done in the initialize function. This is accessible by an instance of the class and only changes withing that instance

Constant Variables

These should not change throughout your program. These start with capital letters and if they are within a class, they are specific to that class.

Basic Literals

There are a few literals that are worth noting.

  • self
  • Booleans
  • Integers
  • Floats
  • Strings
  • Array
  • Hash

If you need to find the scope of a variable, the defined? method takes an object as a parameter and returns its scope.
